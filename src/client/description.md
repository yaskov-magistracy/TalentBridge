# TalentBridge Platform - Angular Prototype

## Описание проекта

Это Angular-версия прототипа платформы **TalentBridge** - платформы для оценки навыков разработчиков через реальные проектные задания. Платформа соединяет начинающих разработчиков (джунов) с работодателями, предоставляя возможность получить реальный опыт через выполнение тестовых заданий.

## Основные возможности

### Для кандидатов (джунов):
- Просмотр доступных заданий от компаний
- Фильтрация заданий по технологиям
- Взятие заданий в работу
- Отправка решений через GitHub
- Отслеживание статуса проверки (автотесты → AI-анализ → экспертное ревью)
- Управление профилем и навыками
- Чат с компаниями

### Для работодателей:
- Создание и редактирование заданий
- Просмотр списка кандидатов и их решений
- Экспертная проверка решений с возможностью одобрения/отклонения
- Рейтинг кандидатов с фильтрацией и сортировкой
- Просмотр детального профиля кандидатов

## Архитектура

### Компоненты

Проект использует **standalone компоненты** Angular (введены в Angular 14+), которые не требуют декларации в NgModule. Это упрощает структуру и делает компоненты более самодостаточными.

#### Общие компоненты (Shared Components)

- **NavbarComponent** - Навигационная панель с логотипом и ролью пользователя
- **FooterComponent** - Подвал с навигацией по разделам
- **TechChipComponent** - Отображение технологии в виде чипа
- **StatusBadgeComponent** - Цветной бейдж статуса проверки
- **ReviewProgressComponent** - Визуализация трехэтапной проверки

#### Страницы (Pages)

Каждая страница представляет собой standalone компонент, который:
- Импортирует необходимые модули (CommonModule, FormsModule, RouterModule)
- Использует shared компоненты
- Содержит свою логику и шаблон

### Данные

Файл `mock-data.ts` содержит:
- **Интерфейсы TypeScript** для типизации данных
- **Mock данные** для демонстрации функциональности

Основные интерфейсы:
- `Task` - задание
- `Submission` - отправленное решение
- `Candidate` - кандидат
- `CandidateRanking` - кандидат в рейтинге

## Технологии

## Установка и запуск

### Требования
- Node.js 18+ 
- npm или yarn

### Установка зависимостей
```bash
cd figma_prototype_angular
npm install
```

### Запуск в режиме разработки
```bash
npm start
# или
ng serve
```

Приложение будет доступно по адресу: http://localhost:4200

### Сборка для продакшена
```bash
npm run build
# или
ng build
```

Сборка будет создана в папке `dist/`.

## Особенности реализации

### Трехэтапная проверка решений

Каждое решение проходит три этапа проверки:

1. **Автотесты** - автоматическая проверка функциональности
2. **AI-анализ** - анализ кода на соответствие best practices
3. **Экспертное ревью** - проверка опытным разработчиком

### Система статусов

- `pending` - ожидание
- `passed` - пройдено
- `failed` - не пройдено
- `approved` - одобрено экспертом
- `rejected` - отклонено экспертом

### Уровни навыков

- `начинающий` - начальный уровень (желтый)
- `базовый` - базовый уровень (индиго)
- `опытный` - опытный уровень (зеленый)

## Дальнейшее развитие

Для превращения прототипа в полноценное приложение необходимо:

1. **Backend API** - реализовать REST API для работы с данными
2. **Аутентификация** - добавить JWT или OAuth2
3. **Состояние** - использовать NgRx или Akita для управления состоянием
4. **Валидация** - добавить валидацию форм с помощью Reactive Forms
5. **Тестирование** - написать unit и e2e тесты
6. **Локализация** - добавить i18n для поддержки нескольких языков
7. **Оптимизация** - добавить lazy loading для модулей
